# krofnePackUpdater

**Обновщик модов для Minecraft 1.20.1 Forge**

---

## Что это за приложение

`krofnePackUpdater` — это десктопное приложение на Electron, которое автоматически синхронизирует моды в папке `.minecraft/mods` с манифестом сборки, опубликованным на GitHub. Приложение:

- Загружает манифест (список модов с хэшами SHA-512 и URL-адресами скачивания) из GitHub
- Сравнивает локальные моды с манифестом
- Скачивает недостающие моды
- Заменяет моды с несовпадающим хэшем
- Перемещает лишние моды в папку `_disabled_by_krofne_pack`
- Создает бэкапы перед каждым обновлением
- Позволяет откатиться на предыдущую версию через бэкапы

Сборка: **krofnePack** для **Minecraft 1.20.1 Forge**.

## Запуск в dev-режиме

### Только фронтенд (Vite, без Electron)

Удобно для разработки UI-компонентов:

```bash
npm run dev
```

Откроется `http://127.0.0.1:5173`. При запуске без Electron API `window.krofnePack` будет недоступен — UI покажет заглушку.

### Полный Electron-режим

```bash
npm run electron:dev
```

Запустит Vite dev-сервер + Electron. В этом режиме доступны все IPC-функции: сканирование папок, загрузка манифеста, синхронизация модов и т.д.

## Сборка Windows installer

```bash
npm run package:win
```

Создаёт NSIS-установщик в папке `release/`. Конфигурация сборки — в `package.json` (поле `build`).

### Как собирается Electron

Electron main и preload **НЕ** собираются через `tsc` emit. Вместо этого:

1. **TypeScript typecheck**: `tsc --noEmit` + `tsc -p tsconfig.node.json --noEmit` — проверка типов без генерации JS
2. **esbuild bundle**: `node scripts/build-electron.mjs` — собирает runtime JS:
   - `electron/main.ts` → `dist-electron/main.js` (ESM)
   - `electron/preload.ts` → `dist-electron/preload.cjs` (CJS)

Почему preload — CJS: Electron preload не поддерживает ESM-модули — падает с `Cannot use import statement outside a module`. esbuild bundle также решает проблему extensionless ESM imports.

### NSIS symlink issue (Windows)

Если `package:win` падает с ошибкой `Cannot create symbolic link`:

1. Включи Windows Developer Mode: **Settings → Privacy & security → For developers → Developer Mode → On**
2. Или запусти PowerShell от администратора
3. Очисти cache: `Remove-Item -Recurse -Force "$env:LOCALAPPDATA\electron-builder\Cache\winCodeSign" -ErrorAction SilentlyContinue`
4. Повтори `npm run package:win`

Fallback: portable ZIP из `release/win-unpacked/`.

Для сборки нужен **Windows** (electron-builder поддерживает кросс-компиляцию, но NSIS-инсталлер создаётся только на Windows).

## Как устроен manifest

Манифест — это JSON-файл, который описывает все моды сборки. Он размещается на GitHub (raw-URL) и скачивается приложением при каждой проверке.

### Схема manifest.json

```json
{
  "schemaVersion": 1,
  "packName": "krofnePack",
  "packVersion": "1.0.0",
  "minecraftVersion": "1.20.1",
  "loader": "forge",
  "githubRawBaseUrl": "https://raw.githubusercontent.com/KROFN/krofne-minecraft-pack/main/mods",
  "settings": {
    "maxParallelDownloads": 3,
    "downloadRetries": 3,
    "downloadTimeoutMs": 60000
  },
  "server": {
    "name": "krofne server",
    "address": "example.com",
    "port": 25565
  },
  "allowedExtraMods": [
    { "pattern": "minimap*.jar", "description": "Любой миникарта" },
    { "pattern": "optifine*.jar", "description": "OptiFine" }
  ],
  "changelog": [
    {
      "version": "1.0.0",
      "date": "2025-03-05",
      "items": [
        "Добавлен Sophisticated Backpacks",
        "Обновлён Create до 0.5.1f"
      ]
    }
  ],
  "mods": [
    {
      "name": "Create",
      "fileName": "create-1.20.1-0.5.1f.jar",
      "sha512": "a1b2c3d4e5f6...полный_хэш_sha512",
      "downloadUrl": "https://raw.githubusercontent.com/KROFN/krofne-minecraft-pack/main/mods/create-1.20.1-0.5.1f.jar",
      "size": 13025280
    }
  ]
}
```

### Обязательные поля

| Поле | Тип | Описание |
|------|-----|----------|
| `schemaVersion` | number | Версия схемы (пока `1`) |
| `packName` | string | Название сборки |
| `packVersion` | string | Версия сборки (semver) |
| `minecraftVersion` | string | Версия Minecraft |
| `loader` | string | Загрузчик: `forge`, `fabric`, `neoforge`, `quilt` |
| `mods` | array | Массив модов |

### Поля мода

| Поле | Тип | Описание |
|------|-----|----------|
| `name` | string | Человекочитаемое название |
| `fileName` | string | Имя .jar-файла |
| `sha512` | string | SHA-512 хэш файла |
| `downloadUrl` | string | Прямая ссылка для скачивания |
| `size` | number | Размер в байтах |

### Где разместить манифест

Манифест размещается в репозитории GitHub. URL по умолчанию:
```
https://raw.githubusercontent.com/KROFN/krofne-minecraft-pack/main/manifest.json
```

Моды размещаются в том же репозитории (папка `mods/`), чтобы GitHub Raw отдавал прямые ссылки на скачивание.

## Как пользоваться обычному игроку

### Простой режим (по умолчанию)

1. **Скачайте и запустите** `krofnePackUpdater.exe`
2. При первом запуске приложение предложит **найти папку mods** автоматически или выбрать вручную
3. После выбора папки нажмите **«Проверить моды»**
4. Если моды актуальны — вы увидите зелёную галочку ✅
5. Если нужно обновление — вы увидите:
   - Сколько модов нужно **скачать**
   - Сколько модов нужно **заменить**
   - Сколько **лишних** модов будет отключено
6. Нажмите **«Синхронизировать»** — приложение:
   - Создаст бэкап текущих модов
   - Скачает недостающие моды
   - Заменит моды с несовпадающим хэшем
   - Перемещёт лишние моды в папку `_disabled_by_krofne_pack`
7. Дождитесь завершения — готово! 🎉

### Подробный режим

Если нужен больший контроль — переключитесь в подробный режим (кнопка в правом верхнем углу). Там доступны:

- **Моды** — таблица со всеми модами, фильтрация по статусу
- **Бэкапы** — список бэкапов с возможностью отката
- **Админ** — генератор манифеста (для владельца сборки)
- **Настройки** — URL манифеста, параллельные загрузки, режим отладки

## Как пользоваться admin mode

### Генерация манифеста

1. Перейдите в раздел **«Админ»** (подробный режим)
2. Заполните параметры:
   - Название сборки
   - Версия сборки
   - Версия Minecraft
   - Загрузчик
   - GitHub Raw базовый URL
   - Путь к папке с модами
3. Нажмите **«Сканировать папку»** — приложение посчитает SHA-512 для каждого .jar
4. Нажмите **«Сгенерировать»** — манифест будет создан
5. Скопируйте манифест или сохраните в файл

### Редактор чейнджлога

В разделе «Админ» → «Редактор чейнджлога» можно добавить записи об изменениях для каждой версии.

## Как обновлять сборку на GitHub

### Workflow для владельца сборки

1. **Подготовьте новые моды:**
   - Скачайте/обновите .jar-файлы модов
   - Положите их в папку `mods/` в репозитории

2. **Обновите манифест:**
   - Запустите krofnePackUpdater
   - Перейдите в Админ → Генератор манифеста
   - Укажите новую версию (например, `1.1.0`)
   - Отсканируйте папку с новыми модами
   - Сгенерируйте манифест
   - Скопируйте JSON

3. **Загрузите на GitHub:**
   - Замените `manifest.json` в репозитории
   - Загрузите новые .jar файлы в папку `mods/`
   - Сделайте commit и push

4. **Игроки получат обновление** при следующей проверке в приложении

### Важно

- Убедитесь, что `downloadUrl` в манифесте указывает на реальный raw-URL GitHub
- GitHub Raw имеет лимит — файлы до ~25 МБ. Для больших модов используйте GitHub Releases
- Не меняйте `fileName` для существующих модов без необходимости — это приведёт к перескачиванию

## Troubleshooting

### Приложение не находит папку mods

**Решение:** Нажмите «Выбрать папку вручную» и укажите путь к `.minecraft/mods`. Стандартные пути:
- Windows: `%APPDATA%\.minecraft\mods`
- macOS: `~/Library/Application Support/minecraft/mods`
- Linux: `~/.minecraft/mods`

### Ошибка «Не удалось загрузить манифест»

**Причины:**
- Нет интернета — проверьте соединение
- URL манифеста неверный — проверьте в Настройках
- Репозиторий GitHub недоступен — попробуйте позже

**Решение:** Проверьте URL в Настройках. По умолчанию:
```
https://raw.githubusercontent.com/KROFN/krofne-minecraft-pack/main/manifest.json
```

### Ошибка «Хэш не совпадает» после скачивания

**Причина:** Файл повредился при скачивании.

**Решение:** Приложение автоматически повторит попытку (до 3 раз). Если ошибка повторяется — проверьте URL загрузки в манифесте.

### Лишние моды — куда они делись?

Лишние моды (которых нет в манифесте и нет в списке разрешённых) перемещаются в папку:
```
.minecraft/mods/_disabled_by_krofne_pack/
```

Моды из этой папки **не загружаются** Minecraft. Вы можете вернуть их вручную, если нужно.

### Как откатиться на предыдущую версию?

1. Переключитесь в подробный режим
2. Перейдите в раздел **«Бэкапы»**
3. Найдите нужный бэкап
4. Нажмите **«Откатить»**
5. Подтвердите действие

Перед откатом приложение создаст экстренный бэкап текущего состояния.

### Синхронизация зависла

**Решение:**
1. Приложение обнаружит прерванную синхронизацию и предложит **Recovery** (Восстановление)
2. Нажмите «Восстановить» — приложение удалит незавершённые `.download` файлы
3. Нажмите «Проверить моды» заново — уже скачанные файлы будут взяты из cache
4. Если приложение не предлагает Recovery: удалите `.krofne-sync-state.json` из папки mods и перезапустите

## Надёжное скачивание (v1.2.0)

Начиная с версии 1.2.0, приложение использует устойчивую систему загрузки:

### Verified Download Cache

- Скачанные и проверенные (SHA-512) файлы сохраняются в папку `mods/_krofne_download_cache/`
- Каждый файл в cache называется по SHA-512 хэшу: `<sha512>.jar`
- Перед скачиванием приложение проверяет cache — если файл уже есть и хэш совпадает, скачивание **не происходит**
- Если cache повреждён (хэш не совпадает) — файл удаляется и скачивается заново

### Не нужно качать всё заново после сбоя

Если синхронизация прервалась (например, на 31-м моде из 40):
- Первые 30 модов уже сохранены в cache и/или установлены
- При повторной синхронизации приложение скачает только оставшиеся 10 модов
- Файлы из cache копируются в `mods/` мгновенно

### Отмена синхронизации

- Во время синхронизации доступна кнопка **«Отменить»**
- При отмене:
  - Текущие загрузки прерываются
  - Незавершённые `.download` файлы удаляются
  - Уже успешно скачанные файлы остаются в cache
  - Состояние синхронизации → `cancelled`
- После отмены можно нажать «Проверить моды» и продолжить — уже скачанное не перескачивается

### Прогресс загрузки

- Прогресс-бар обновляется в реальном времени (не только 0% → 100%)
- Показывается текущий файл и его прогресс скачивания
- Для каждого файла отображается процент и размер
- Общий прогресс считает по байтам, если в манифесте указан `sizeBytes`

### Timeout

- Timeout одного файла — **30 минут** (было 60 секунд)
- При timeout показывается понятное сообщение: «Скачивание заняло слишком много время. Проверь интернет и попробуй снова.»
- Технические детали доступны в режиме отладки

### Понятные ошибки

Все ошибки нормализуются для пользователя:

| Ситуация | Заголовок | Сообщение |
|----------|-----------|----------|
| Timeout | Скачивание заняло слишком много времени | Проверь интернет и попробуй снова |
| Нет интернета | Проблема с интернетом | Не удалось скачать файл. Проверь подключение |
| 404 | Файл не найден | Сообщи владельцу сборки |
| SHA-512 не совпадает | Файл скачался повреждённым | Попробуй снова или сообщи владельцу |
| Нет прав | Нет прав на запись | Выбери другую папку или запусти от администратора |
| Отмена | Синхронизация отменена | Уже скачанные файлы сохранены |

### Очистка cache

- В Settings → «Cache загрузок» → «Очистить cache»
- Удаляется только папка `_krofne_download_cache`
- Это безопасно — при следующей синхронизации файлы скачаются заново
- Cache **не** удаляется при Recovery

### Служебные папки (не удалять!)

| Папка | Назначение |
|-------|------------|
| `_krofne_download_cache/` | Проверенные скачанные файлы (cache) |
| `_backup_by_krofne_pack/` | Бэкапы перед заменой модов |
| `_disabled_by_krofne_pack/` | Отключённые лишние моды |

### Где хранятся логи?

Логи хранятся в папке `userData` приложения:
- Windows: `%APPDATA%\krofnePackUpdater\logs\`
- macOS: `~/Library/Logs/krofnePackUpdater/`
- Linux: `~/.config/krofnePackUpdater/logs/`

В приложении логи можно посмотреть в разделе «Логи» (подробный режим) или скопировать через кнопку «Копировать».
