- ☐ Автоопределение `%APPDATA%\.minecraft\mods`  
  **Шаги:**
  1. Убедиться, что `%APPDATA%\.minecraft\mods` существует и содержит `.jar` файлы

  **Ожидаемый результат:** В списке кандидатов появляется `.minecraft/mods` с confidence: `high`, reason: «Standard Minecraft Forge mods folder found».

- ☐ Кандидат «.minecraft (create mods folder)»  
  **Шаги:**
  1. Удалить/переименовать папку `mods` внутри `.minecraft`
  2. Запустить приложение
  **Ожидаемый результат:** Появляется кандидат с confidence: `medium`, reason: «Minecraft folder found; mods subfolder can be created».

### 3.2 Ручной выбор папки

- ☐ Выбор папки через диалог (кнопка «Выбрать папку»)  
  **Ожидаемый результат:** Открывается системный диалог Windows «Select Minecraft mods folder». После выбора путь сохраняется в `lastModsPath`.

### 3.3 Путь с пробелами

- ☐ Выбор пути, содержащего пробелы  
  **Шаги:**
  1. Создать папку `C:\My Games\Minecraft Mods\mods`
  2. Поместить туда тестовые `.jar` файлы
  3. Выбрать эту папку в приложении
  4. Выполнить проверку (check)
  **Ожидаемый результат:** Папка корректно сканируется, моды определяются, sync-план строится без ошибок. Пути корректно нормализуются через `normalizePath`.

### 3.4 Путь с кириллицей

- ☐ Выбор пути, содержащего кириллические символы  
  **Шаги:**
  1. Создать папку `C:\Игры\Майнкрафт\mods`
  2. Поместить туда тестовые `.jar` файлы
  3. Выбрать эту папку в приложении
  4. Выполнить проверку (check)
  **Ожидаемый результат:** Папка корректно сканируется. Моды отображаются, синхронизация работает. Имена файлов с кириллицей не вызывают ошибок.

### 3.5 Отсутствующая папка mods

- ☐ Выбор несуществующей папки  
  **Шаги:**
  1. Указать путь к папке, которая не существует (например, через сохранение settings.json вручную)
  2. Попытаться выполнить `sync:check`
  **Ожидаемый результат:** Ошибка: «Mods folder does not exist: <путь>». Приложение не падает.

### 3.6 Нет прав на запись

- ☐ Папка только для чтения  
  **Шаги:**
  1. Создать папку `mods`
  2. Установить права «Только чтение» (Свойства папки → Только чтение)
  3. Попытаться выполнить sync
  **Ожидаемый результат:** Ошибка: «Нет прав на запись в папку mods: <путь>». Sync не начинается. Файлы не изменяются.

## 4. Проверка manifest

### 4.1 Корректный URL

- ☐ Загрузка по умолчанию `https://raw.githubusercontent.com/KROFN/krofne-minecraft-pack/main/manifest.json`  
  **Ожидаемый результат:** Manifest загружен. Отображаются: `packName`, `packVersion`, `minecraftVersion` = `1.20.1`, `loader` = `forge`. Список модов непустой.

### 4.2 Неработающий URL

- ☐ Указать URL, который возвращает 404  
  **Шаги:**
  1. В Settings → `manifestUrl` = `https://raw.githubusercontent.com/KROFN/krofne-minecraft-pack/main/nonexistent.json`
  2. Попробовать загрузить manifest
  **Ожидаемый результат:** Ошибка загрузки. Приложение не падает. Ошибка показана пользователю. Повторная попытка не делается (404 — non-retryable).

- ☐ Указать URL, который не существует (DNS ошибка)  
  **Шаги:**
  1. `manifestUrl` = `https://nonexistent-domain-12345.com/manifest.json`
  2. Попробовать загрузить manifest
  **Ожидаемый результат:** Ошибка `ENOTFOUND` / network error. Отображается понятное сообщение об ошибке сети.

### 4.3 Невалидный JSON

- ☐ URL возвращает не JSON  
  **Шаги:**
  1. Настроить URL, возвращающий HTML или простой текст
  2. Попробовать загрузить manifest
  **Ожидаемый результат:** Ошибка парсинга JSON. Понятное сообщение об ошибке.

### 4.4 Manifest без обязательных полей

- ☐ Manifest без `schemaVersion`  
  **Ожидаемый результат:** Ошибка валидации: обязательное поле отсутствует.

- ☐ Manifest без `packName`  
  **Ожидаемый результат:** Ошибка валидации.

- ☐ Manifest без `packVersion`  
  **Ожидаемый результат:** Ошибка валидации.

- ☐ Manifest без `minecraftVersion`  
  **Ожидаемый результат:** Ошибка валидации: `Manifest is missing or has invalid "minecraftVersion".`

- ☐ Manifest без `loader`  
  **Ожидаемый результат:** Ошибка валидации: `Manifest is missing or has invalid "loader".`

- ☐ Manifest с `loader` = неизвестное значение (не forge/fabric/neoforge/quilt)  
  **Ожидаемый результат:** Ошибка валидации: `Manifest "loader" must be one of: forge, fabric, neoforge, quilt. Got: "<значение>".`

- ☐ Manifest с пустым массивом `mods`  
  **Ожидаемый результат:** Manifest загружен, но sync-план пустой (0 действий).

- ☐ Manifest с `minecraftVersion` ≠ `1.20.1`  
  **Ожидаемый результат:** Manifest загружен. При проверке/sync — предупреждение или ошибка о несовпадении версии Minecraft (поведение зависит от UI).

### 4.5 SHA-512 несовпадение в manifest

- ☐ Manifest содержит неверный SHA-512 для одного из модов  
  **Шаги:**
  1. Подготовить manifest, в котором `sha512` одного из модов не совпадает с реальным хешем файла на сервере
  2. Запустить sync
  **Ожидаемый результат:** Скачивание завершается, но hash-verification fails. Файл `.download` удаляется. Попытка retry (hash mismatch — retryable). После исчерпания всех 3 попыток — sync прерывается, состояние → `interrupted`.

## 5. Проверка sync на копии mods

> **ВАЖНО:** Перед каждым тестом делайте **полную копию** папки mods.  
> Не тестировать на реальной папке с модами!

**Подготовка:** Создать тестовую папку `mods` с файлами:
- `already-correct.jar` — .jar, чей SHA-512 совпадает с manifest
- `old-version-mod.jar` — .jar, чьё имя совпадает с manifest, но SHA-512 отличается
- `unknown-extra.jar` — .jar, отсутствующий в manifest и не подходящий под `allowedExtraMods`
- `allowed-minimap.jar` — .jar, отсутствующий в manifest, но подходящий под правило `allowedExtraMods` (например, `filename_contains: "minimap"`)

Manifest должен содержать:
- Мод с SHA-512 = хешу `already-correct.jar` → категория `installed`
- Мод с fileName = `old-version-mod.jar`, но SHA-512 ≠ хешу файла → категория `wrong_hash`
- Мод, которого нет в папке → категория `missing`

### 5.1 installed

- ☐ Мод, уже соответствующий manifest, остаётся без изменений  
  **Ожидаемый результат:** В sync-плане мод в категории `installed`. После sync файл не перемещается, не перескачивается. Размер и SHA-512 неизменны.

### 5.2 missing

- ☐ Отсутствующий мод скачивается  
  **Ожидаемый результат:** Мод в категории `missing`. После sync файл `.jar` появляется в папке mods. SHA-512 совпадает с manifest. Временный файл `.download` удалён (переименован в `.jar`).

### 5.3 wrong_hash

- ☐ Мод с неверным хешем заменяется  
  **Ожидаемый результат:** Мод в категории `wrong_hash`. Старый файл скопирован в `_backup_by_krofne_pack/backup_YYYY-MM-DD_HH-mm-ss/` **до** удаления. Старый файл удалён из `mods/`. Новый `.jar` скачан и верифицирован.

### 5.4 extra

- ☐ Неизвестный мод перемещается в `_disabled_by_krofne_pack`  
  **Ожидаемый результат:** Мод в категории `extra`. Файл перемещён (не скопирован!) из `mods/` в `mods/_disabled_by_krofne_pack/unknown-extra.jar`. Файл **не удалён**.

- ☐ При конфликте имён в `_disabled_by_krofne_pack` добавляется timestamp  
  **Шаги:**
  1. Поместить файл с таким же именем в `_disabled_by_krofne_pack` вручную
  2. Запустить sync
  **Ожидаемый результат:** Файл в disabled folder получает суффикс: `unknown-extra__YYYY-MM-DD_HH-mm-ss.jar`.

### 5.5 allowed_extra

- ☐ Разрешённый extra-мод остаётся на месте  
  **Шаги:**
  1. В manifest добавить правило: `allowedExtraMods: [{ match: "filename_contains", value: "minimap", reason: "Player choice" }]`
  2. В папке `mods/` лежит `allowed-minimap.jar`
  **Ожидаемый результат:** Мод в категории `allowed_extra`. Файл **не перемещается** в `_disabled_by_krofne_pack`, остаётся в `mods/`.

- ☐ Правило `filename_regex` работает  
  **Шаги:**
  1. Правило: `{ match: "filename_regex", value: ".*minimap.*", reason: "Regex test" }`
  2. В папке `allowed-minimap.jar`
  **Ожидаемый результат:** Мод классифицирован как `allowed_extra`.

- ☐ Правило `sha512` работает  
  **Шаги:**
  1. Правило: `{ match: "sha512", value: "<точный_хеш_файла>", reason: "Exact hash test" }`
  2. Файл с этим хешем в `mods/`
  **Ожидаемый результат:** Мод классифицирован как `allowed_extra`.

### 5.6 backup

- ☐ Файлы с wrong_hash бэкапятся **до** замены  
  **Шаги:**
  1. Запустить sync с wrong_hash модом
  2. Проверить папку `mods/_backup_by_krofne_pack/backup_YYYY-MM-DD_HH-mm-ss/`
  **Ожидаемый результат:** В backup-папке лежит копия старого файла. Имя файла совпадает с оригинальным.

- ☐ `backup-meta.json` создан корректно  
  **Ожидаемый результат:** Файл `backup-meta.json` в папке backup-сессии содержит:
  - `createdAt` — ISO date
  - `packVersionBefore` — версия до sync (или null)
  - `packVersionAfter` — версия после sync
  - `modsDir` — путь к папке mods
  - `files[]` — массив с `originalPath`, `backupPath`, `fileName`, `reason`

### 5.7 disabled (моды не удаляются)

- ☐ Extra-моды перемещены, а не удалены  
  **Ожидаемый результат:** После sync все extra-моды физически существуют в `mods/_disabled_by_krofne_pack/`. В папке `mods/` их нет. Ничего не удалено безвозвратно.

### 5.8 Нет безвозвратного удаления

- ☐ Проверить, что за весь sync ни один файл не удалён безвозвратно  
  **Шаги:**
  1. Записать список всех файлов в `mods/` и подпапках до sync
  2. Выполнить sync
  3. Записать список всех файлов после sync (включая `_backup_by_krofne_pack/` и `_disabled_by_krofne_pack/`)
  **Ожидаемый результат:** Каждый файл, существовавший до sync, либо остался в `mods/`, либо находится в `_backup_by_krofne_pack/`, либо в `_disabled_by_krofne_pack/`. Ничего не потеряно.

## 6. Проверка recovery

### 6.1 Прерванный sync — закрытие приложения

- ☐ Закрыть приложение во время загрузки модов  
  **Шаги:**
  1. Начать sync с несколькими missing-модами
  2. Пока идёт загрузка (фаза «downloading»), закрыть окно приложения (Alt+F4)
  **Ожидаемый результат:** Приложение закрывается. В папке `mods/` остаётся файл `.krofne-sync-state.json` со `status: "running"` или `"interrupted"`. Остаются временные файлы `.download`.

### 6.2 Проверка .krofne-sync-state.json

- ☐ Файл `.krofne-sync-state.json` существует после прерывания  
  **Шаги:**
  1. Открыть папку `mods/` в проводнике
  2. Найти файл `.krofne-sync-state.json`
  3. Открыть в текстовом редакторе
  **Ожидаемый результат:** Файл содержит JSON с полями: `syncId`, `startedAt`, `status` (`"running"` или `"interrupted"`), `completedActions[]`, `pendingActions[]`, `tempFiles[]`.

### 6.3 Проверка .download temp файлов

- ☐ Временные файлы `.download` существуют  
  **Шаги:**
  1. Открыть папку `mods/` в проводнике
  2. Найти файлы с расширением `.download`
  **Ожидаемый результат:** Присутствуют файлы вида `modname.jar.download` — это незавершённые загрузки.

### 6.4 Выполнение recovery

- ☐ Запустить приложение после прерывания  
  **Ожидаемый результат:** Приложение обнаруживает interrupted sync и предлагает выполнить recovery (или показывает кнопку/уведомление).

- ☐ Нажать «Восстановить» / «Recovery»  
  **Ожидаемый результат:**
  1. Все `.download` файлы удалены из `mods/`
  2. Файл `.krofne-sync-state.json` удалён
  3. Сообщение: «Recovery completed. Please re-check mods to get a fresh sync plan.»

### 6.5 Проверка очистки после recovery

- ☐ После recovery нет `.download` файлов  
  **Ожидаемый результат:** В папке `mods/` нет файлов с расширением `.download`.

- ☐ После recovery нет `.krofne-sync-state.json`  
  **Ожидаемый результат:** Файл `mods/.krofne-sync-state.json` не существует.

### 6.6 Повторная проверка и sync после recovery

- ☐ Выполнить «Проверить моды» после recovery  
  **Ожидаемый результат:** Строится новый sync-план. Нет ошибки о прерванном sync.

- ☐ Выполнить sync после recovery

## Известные ограничения

1. **Только Forge 1.20.1** — Приложение не поддерживает Fabric, NeoForge, Quilt или другие версии Minecraft (несмотря на то, что тип `LoaderType` определяет `forge | fabric | neoforge | quilt`).
2. **Один manifest URL** — Можно настроить только один URL manifest. Переключение между разными модпаками требует ручной смены URL.
3. **Нет增量ной загрузки** — При wrong_hash весь файл скачивается заново, патчинг не поддерживается.
4. **Нет автоматического обновления приложения** — Проверка обновлений реализована (`app:checkUpdate`), но автообновление не встроено. Пользователь должен скачать новый `.exe` вручную.
5. **Windows только** — NSIS installer собирается только для Windows. macOS/Linux не поддерживаются в текущей конфигурации.
6. **Нет proxy-настроек** — Загрузка через HTTP proxy не настраивается. Используется системный proxy ОС.
7. **Timeout 60 секунд** — `DOWNLOAD_TIMEOUT_MS = 60000`. На очень медленном соединении загрузка больших модов может не успеть.
8. **Нет возобновления загрузки** — Прерванная загрузка `.download` файла начинается заново, resume не поддерживается.
9. **Sidebar/TopBar UI** — При ширине окна < 800px (minWidth) часть UI может обрезаться.

---

## Что НЕ тестировать

1. **Автоматические обновления приложения** — Функция `checkForUpdate` проверяет, но не устанавливает обновления автоматически.
2. **Fabric/NeoForge/Quilt совместимость** — Приложение предназначено для Forge 1.20.1; тестирование других загрузчиков вне scope.
3. **macOS/Linux** — Сборка и запуск на других платформах не поддерживаются в текущей конфигурации `package:win`.
4. **Моды с не-.jar расширением** — Сканер ищет только `MOD_FILE_EXTENSION = '.jar'`. Файлы `.zip`, `.disabled` и т.д. игнорируются.
5. **Многопользовательский режим** — Приложение однопользовательское. Нет авторизации, профилей, или разделения данных между пользователями ОС.
6. **Серверную часть Minecraft** — Приложение синхронизует клиентские моды. Серверная установка Forge/mods не тестируется.
7. **Полный pentest Electron** — не нужен. Но перед релизом следует выполнить **sanity-check** безопасности (см. Release checklist).

## Инструкция по экстренному восстановлению для друга

> Если что-то пошло не так при синхронизации, отправь эту инструкцию другу.

### Ситуация 1: Приложение зависло / закрылось во время sync

**Симптомы:** При повторном запуске приложение пишет о прерванной синхронизации.

**Решение:**
1. Открой krofnePackUpdater
2. Приложение предложит выполнить **Recovery** (Восстановление)
3. Нажми кнопку «Восстановить»
4. После завершения нажми «Проверить моды» заново
5. Нажми «Синхронизировать»

### Ситуация 2: Моды пропали из папки mods

**Симптомы:** После sync в папке `mods/` меньше файлов, чем ожидалось.

**Решение:**
1. Открой папку `mods/` в проводнике
2. Проверь папку `mods/_disabled_by_krofne_pack/` — там лежат «лишние» моды, которые были перемещены, а **не удалены**
3. Если хочешь вернуть мод обратно — просто перенеси `.jar` файл из `_disabled_by_krofne_pack/` обратно в `mods/`

### Ситуация 3: Хочу вернуть старую версию мода

**Решение:**
1. Открой krofnePackUpdater
2. Перейди на вкладку **«Backups»** (Резервные копии)
3. Выбери нужную backup-сессию (по дате)
4. Нажми **«Rollback»** (Откатить)
5. Подтверди действие в диалоге
6. Старые версии модов будут восстановлены из backup

### Ситуация 4: Ничего не помогло — ручное восстановление

1. Открой папку `mods/` в проводнике
2. Открой подпапку `_backup_by_krofne_pack/`
3. Найди самую свежую папку `backup_YYYY-MM-DD_HH-mm-ss/`
4. Скопируй `.jar` файлы из этой папки обратно в `mods/`
5. Удали файл `mods/.krofne-sync-state.json` (если существует)
6. Удали все файлы `.download` в папке `mods/` (если есть)
7. Запусти krofnePackUpdater и нажми «Проверить моды»

### Ситуация 5: Полная пересборка с нуля

1. Удали всё содержимое папки `mods/` (кроме подпапок `_backup_by_krofne_pack/` и `_disabled_by_krofne_pack/`)
2. Удали файл `mods/.krofne-sync-state.json` (если существует)
3. Запусти krofnePackUpdater
4. Нажми «Проверить моды» → все моды будут в категории `missing`
5. Нажми «Синхронизировать» → все моды скачаются заново

### Ключевые файлы и папки

| Файл / Папка | Описание |
|---|---|
| `mods/_backup_by_krofne_pack/` | Все резервные копии (не удалять!) |
| `mods/_disabled_by_krofne_pack/` | Отключённые «лишние» моды (не удалены!) |
| `mods/.krofne-sync-state.json` | Файл состояния sync (скрытый) |
| `mods/backup-meta.json` | Метаданные резервной копии (внутри backup-сессии) |
| `%APPDATA%\krofnePackUpdater\settings.json` | Настройки приложения |

### Важно знать

- **Ничего не удаляется безвозвратно.** Все заменённые моды — в `_backup_by_krofne_pack/`, все «лишние» — в `_disabled_by_krofne_pack/`.
- **Rollback создаёт emergency backup.** Даже откат можно откатить.
- **Файлы `.download` — временные.** Их можно безопасно удалить.
- **`.krofne-sync-state.json` — скрытый файл.** В проводнике включи «Показывать скрытые файлы».
